# ratatoskr Design

## 1. 概要

`ratatoskr` は、ローカルフォルダだけでなく S3 などのリモートストレージも参照できるマルチエクスプローラ＋ビューアである。

主な用途は次のとおり。

- ローカルのファイルを階層でたどる
- S3 バケットやプレフィックスを同じ操作感で参照する
- Markdown、画像、テキストなどを即座に確認する
- ファイル一覧とプレビューを並べて作業する

実装は Vue / TypeScript を前提にする。

## 2. 目的

- ローカルとクラウドストレージの差を意識せずに探索できるようにする
- ファイルを開くまでの手順を減らし、確認作業を高速化する
- 将来の接続先追加に備え、ストレージ種別を拡張しやすくする

## 3. 非目的

- 高度な同期機能
- ファイル編集のフル機能
- 大規模な DAM / PIM のような資産管理機能
- S3 以外の全クラウドを最初から網羅すること

## 4. 想定ユーザー

- ローカルファイルと S3 を行き来しながら内容を確認したい開発者
- Markdown や画像をすぐ確認したい運用担当者
- 複数の保存先を 1 つの UI で扱いたい利用者

## 5. コア機能

### 5.1 エクスプローラ

- ローカルフォルダの参照
- S3 バケット / prefix の参照
  対象とするプロファイルの選択
- ツリー表示
- パス入力による直接移動
- 直近の参照先履歴
- 更新ボタンによる再読み込み

### 5.2 ビューア

- Markdown のレンダリング表示
- 画像の表示
- テキストの表示
- 大きいファイルでも最低限のプレビューを維持

### 5.3 共通操作

- ファイル一覧からの選択
- ダブルクリックまたは Enter でプレビュー
- パンくず表示
- 検索 / フィルタ
- 表示モード切り替え

## 6. 画面構成

### 6.1 基本レイアウト

基本レイアウトは 3 ペインとする。

- 上部: ヘッダー
- 下部左: ソースタブ / ナビゲーション
- 下部右: 本文エリア

下部は左右 2 分割とし、本文エリアの中で一覧とビューアを扱う。

### 6.2 ヘッダー

ヘッダーは任意表示とし、必要ない場合は非表示にできる。

表示する場合の主な項目は次のとおり。

- AWS プロファイル選択
- リージョン選択
- 接続状態
- 更新ボタン
- グローバル検索

### 6.3 ソースタブ

ソースはタブで切り替える。

- `Local`
- `S3`
- 将来的な拡張ソース

タブは「接続先の種類」を表し、その中で個別の接続先や保存済み設定を選ぶ。

### 6.4 下部左ペイン

左ペインはナビゲーションに集中する。

- ソース種別タブ
- 接続先一覧
- お気に入り
- 履歴
- 現在位置のパンくず

### 6.5 下部右ペイン

右ペインは本文エリアとする。

- ファイルツリー
- ファイル一覧
- Markdown プレビュー
- 画像プレビュー
- テキストプレビュー
- メタ情報

一覧とビューアは同時表示を基本にし、幅が足りない場合は縦方向の切り替えを許可する。

### 6.6 モバイル / 狭幅時

狭い画面では 3 ペインを 1 画面に詰め込まず、以下のいずれかに切り替える。

- ヘッダー + 2 ペイン
- タブ切り替え
- スライドオーバー表示

## 7. 情報モデル

### 7.1 共通ノード

ストレージ種別に依存しない共通ノードを定義する。

```ts
type NodeKind = "folder" | "file";

interface ExplorerNode {
  id: string;
  name: string;
  kind: NodeKind;
  path: string;
  sourceId: string;
  modifiedAt?: string;
  size?: number;
  mimeType?: string;
}
```

### 7.2 ソース

ソースは参照先の種類を表す。

```ts
type SourceType = "local" | "s3";

interface Source {
  id: string;
  type: SourceType;
  label: string;
  rootPath?: string;
  bucket?: string;
  prefix?: string;
}
```

### 7.3 プレビュー対象

ビューアは選択ノードに応じて表示形式を決める。

```ts
type PreviewKind = "markdown" | "image" | "text" | "unsupported";
```

## 8. データ取得方針

### 8.1 ローカル

- OS のファイルシステム API を利用する
- ディレクトリ走査は遅延読み込みにする
- 大きな階層は必要なノードだけ展開する

### 8.2 S3

- バケット一覧、prefix 一覧、オブジェクト一覧を分けて扱う
- 一覧取得はページングを前提にする
- ファイル本体はプレビュー時に取得する
- 認証情報はアプリ設定として保持する

### 8.3 抽象化

ストレージ差分は UI から隠し、以下のインターフェースに寄せる。

```ts
interface ExplorerProvider {
  listChildren(nodeId: string): Promise<ExplorerNode[]>;
  readPreview(nodeId: string): Promise<Blob | string>;
  getMetadata(nodeId: string): Promise<Record<string, unknown>>;
}
```

## 9. ビューア仕様

### 9.1 Markdown

- GitHub 風の見た目を基本にする
- 見出し、リスト、コードブロック、表を表示する
- 画像埋め込みやリンクを許可する
- 長文でもスクロールしやすいレイアウトにする

### 9.2 画像

- 拡大縮小
- 実寸表示
- 画面に合わせる
- 縦横比を維持

### 9.3 テキスト

- 等幅フォント表示
- 行番号表示は任意
- 改行を保持

## 10. 状態管理

### 10.1 必要な状態

- 選択中のソース
- 現在のパス
- 展開済みツリー
- 選択ファイル
- ビューア種別
- 読み込み中状態
- エラー状態
- 履歴

### 10.2 画面遷移の基本

1. ソースを選ぶ
2. ルートまたは指定パスを開く
3. 一覧からファイルを選ぶ
4. プレビューが右ペインに表示される

## 11. エラーハンドリング

- 接続失敗はソース単位で表示する
- 一覧取得失敗は対象パスに紐づけて表示する
- プレビュー失敗はビューア内で明示する
- 再試行ボタンを用意する

## 12. 拡張ポイント

- Azure Blob など別ストレージの追加
- ファイル編集機能
- お気に入りフォルダの永続化
- 最近開いたファイルの保存
- ファイル比較機能

## 13. 技術方針

- フロントエンドは Vue + TypeScript
- UI はコンポーネント分割を前提にする
- ストレージ別の実装は provider パターンで分離する
- ビューアはファイル種別ごとに renderer を分ける

## 14. 初期実装の優先順位

1. ローカルフォルダの参照
2. ファイル一覧とプレビュー
3. Markdown と画像の表示
4. S3 接続
5. 履歴 / お気に入り
6. 検索 / フィルタ

## 15. 確認したい点

- S3 は AWS S3 を最初の対象にするか
- ファイルの編集機能も将来入れるか
- 3 ペイン固定にするか、狭幅時の切り替え方法をどうするか
- ビューアで Markdown の外部リンクをアプリ内遷移にするか既定ブラウザに送るか
